# vscode-hatenablog

ローカルのMarkdownをはてなブログAPIを使って記事投稿するVSCode拡張機能。

## 1. 全体方針・コミュニケーション

- ユーザーは日本人。コード内コメント・出力メッセージ・質問は日本語で行う。
- 既存のコードコメントは、明示的な指示がない限り変更しない。
- テストで実行してよいのは `npm run test:unit`（単体テスト）のみ。npm run test,integration（統合テスト）は実行禁止。
- API仕様は `api.md` を参照。

## 2. 技術スタック

- 言語: TypeScript
- Lint: biome v2

## 3. TypeScript / コーディングスタイル

- 型定義は `interface` ではなく `type` を使う。
- `any` 型は絶対に使用しない。
- 型アサーション `as` は原則使用しない。やむを得ず使う場合はコードコメントで理由を説明する。
- `class` 構文は一切使用しない。
- 関数定義はすべてアロー関数を使用する。
- 条件分岐は早期リターンを用いてフラットに保つ。
- `try-catch` は乱用せず、必要最低限のみ使用する。

## 4. frontmatter仕様

Markdownファイルの先頭に以下のフォーマットでメタデータを記述する。

```md
---
id: 17179246901363868670
title: "投稿テスト"
date: "2026-03-11"
categories: ["ubuntu"]
published_at: "2026-03-11T23:47:27+09:00"
updated_at: "2026-03-11T23:47:27+09:00"
draft_flag: false
---

# 本文開始

本文はマークダウン形式だよ。
```
